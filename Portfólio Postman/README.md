 Descrição do Projeto

Este projeto consiste na execução de testes manuais em uma API REST, utilizando o Postman, com foco na validação de regras de negócio, 
contratos básicos e comportamento da API em cenários positivos e negativos.
O projeto foi desenvolvido sem automação, priorizando o raciocínio analítico e a compreensão do funcionamento da API.
.
.
.
   Objetivo

Validar endpoints da API
Garantir que regras de negócio sejam respeitadas
Analisar respostas, status codes e mensagens de erro
Demonstrar pensamento crítico em testes de API
.
.
.

  Escopo dos Testes

Cadastro de usuários
Login
Edição de usuários
Pesquisa por ID
Validação de dados inválidos
Cenários negativos e de borda
.
.
.

  Estratégia de Testes

Testes manuais de API
Validação de Status Codes HTTP
Análise de mensagens de erro
Validação de comportamento inesperado (ex: edição com dados em branco)
Atenção ao impacto de falhas para o sistema e negócio
.
.
.

  Artefatos Produzidos

Casos de Teste de API
Evidências de execução (prints do Postman)
Collection do Postman em formato JSON
Documentação dos cenários testados
.
.
.
  Ferramentas Utilizadas

Postman
API REST
JSON
Google Sheets / Excel
.
.
.
  Conclusão

Este projeto demonstra a evolução do pensamento QA, indo além da execução de requisições, com foco em análise de regras de negócio, inconsistências e riscos.
