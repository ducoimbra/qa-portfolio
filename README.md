  Como penso como QA

Meu foco não é apenas validar se “funciona”, mas questionar:
E se o usuário fizer isso?
Essa regra de negócio faz sentido?
Qual o impacto dessa falha para o sistema ou negócio?
O comportamento da API está coerente com o contrato?

Acredito que qualidade vem do pensamento analítico humano, e não apenas da execução mecânica de testes.
